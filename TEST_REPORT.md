**입력값 검증** (9/9)
- ✅ 비어있는 이메일
- ✅ 잘못된 이메일 형식
- ✅ 약한 비밀번호
- ✅ 잘못된 휴대폰 형식

- ✅ partner_staff의 partner_id 필수
- ✅ 비어있는 로그인 필드
- ✅ 존재하지 않는 사용자
- ✅ 잘못된 비밀번호

**SQL Injection 방지** (4/4)
- ✅ `'; DROP TABLE users; --`
- ✅ `1' OR '1'='1`
- ✅ `admin'--`
- ✅ `1 UNION SELECT * FROM users`

**인증 검증** (3/3)
- ✅ 토큰 없이 주문 생성 시도 → 401
- ✅ 잘못된 토큰 사용 → 401
- ✅ 토큰 없이 파트너 조회 → 401

#### 개선 필요 항목 ⚠️

**XSS 방지** (0/3)
- ❌ `<script>alert("XSS")</script>` - 회원가입 성공 (201)
- ❌ `<img src=x onerror="alert(1)">` - 회원가입 성공 (201)
- ❌ `javascript:alert(1)` - 회원가입 성공 (201)

**권장사항**:
- 서버단에서 HTML 특수문자 escaping 추가
- 입력값 sanitization 라이브러리 도입 (e.g., `xss` npm package)
- 클라이언트에서는 이미 브라우저 XSS 방지 기능으로 보호됨
